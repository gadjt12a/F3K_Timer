# Release Firmware

Package a versioned firmware release for the F3K Timer (waveshare env).

## Steps — execute in order, stop and report if any step fails

### 1. Determine next version

```powershell
git tag --list "fw-v*" | Sort-Object { [int]($_ -replace 'fw-v','') } | Select-Object -Last 1
```

If no tags exist, start at `fw-v1`. Otherwise increment by 1 (e.g. `fw-v11` → `fw-v12`).
Store as `$version` (e.g. `fw-v12`).

### 2. Write fw_version.h BEFORE building

Use the Write tool to write `include/fw_version.h`:

```
// Auto-generated by release-firmware skill -- do not edit manually
#pragma once
#define FW_VERSION "<version>"
```

Replace `<version>` with the actual version string (e.g. `fw-v12`).

### 3. Build waveshare firmware

```powershell
& "$env:USERPROFILE\.platformio\penv\Scripts\pio.exe" run -e waveshare --project-dir "C:\Kris\Projects\F3K_Timer_1" 2>&1 | Select-Object -Last 30
```

A successful build ends with `[SUCCESS]`. If it fails, stop and show the last 50 lines of output.

### 4. Verify build outputs exist

Check that all three files are present:
- `.pio/build/waveshare/firmware.bin`
- `.pio/build/waveshare/bootloader.bin`
- `.pio/build/waveshare/partitions.bin`

```powershell
@(".pio/build/waveshare/firmware.bin", ".pio/build/waveshare/bootloader.bin", ".pio/build/waveshare/partitions.bin") | ForEach-Object { Test-Path $_ }
```

All three must return True. If any is missing, stop.

### 5. Create release folder and copy binaries

```powershell
New-Item -ItemType Directory -Force "firmware\releases\$version"
Copy-Item ".pio\build\waveshare\firmware.bin"    "firmware\releases\$version\firmware.bin"
Copy-Item ".pio\build\waveshare\bootloader.bin"  "firmware\releases\$version\bootloader.bin"
Copy-Item ".pio\build\waveshare\partitions.bin"  "firmware\releases\$version\partitions.bin"
```

### 6. Write release.txt

Get the current HEAD commit hash:
```powershell
git rev-parse HEAD
```

Use the Write tool to write `firmware/releases/<version>/release.txt`:

```
F3K Timer <version>
Date: <today YYYY-MM-DD>
Source commit: <HEAD hash>

Flash addresses (esptool):
  0x00000  bootloader.bin
  0x08000  partitions.bin
  0x10000  firmware.bin

esptool command:
  esptool.py --chip esp32s3 --port COM4 --baud 921600 write_flash 0x0 bootloader.bin 0x8000 partitions.bin 0x10000 firmware.bin
```

### 7. Trim releases to 5 most recent

List all `firmware/releases/fw-v*` directories sorted by version number. If more than 5 exist after adding the new one, delete the oldest.

```powershell
$dirs = Get-ChildItem "firmware\releases" -Directory | Where-Object { $_.Name -match "^fw-v\d+$" } | Sort-Object { [int]($_.Name -replace 'fw-v','') }
if ($dirs.Count -gt 5) {
    $dirs | Select-Object -First ($dirs.Count - 5) | ForEach-Object { Remove-Item $_.FullName -Recurse -Force }
}
```

### 8. Update firmware/ota/

```powershell
Copy-Item ".pio\build\waveshare\firmware.bin" "firmware\ota\firmware.bin" -Force
```

Use the Write tool to write `firmware/ota/version.json`:
```json
{"version":"<version>","url":"http://192.168.10.1:8080/ota/firmware.bin"}
```

### 9. Stage and commit

```powershell
git add "firmware/releases/$version" "firmware/ota/firmware.bin" "firmware/ota/version.json" "include/fw_version.h"
```

Commit with a message following the project pattern:
```powershell
git commit -m @'
Release <version> -- compiled waveshare firmware snapshot (<HEAD hash>)

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
'@
```

Then tag:
```powershell
git tag <version>
```

### 10. Report and prompt for push

Print a summary:
```
Released <version>
  firmware/releases/<version>/  -- binary snapshot
  firmware/ota/                 -- updated for base station OTA
  git tag: <version>

To publish:
  git push; git push origin <version>
```

Do NOT push automatically. Wait for the user to confirm.

---

## Critical rules

- **Never skip the build** (step 3). `fw_version.h` must be written before building or the
  embedded `FW_VERSION` string won't match the tag — the OTA check will loop forever.
- **PowerShell 5.1 only** — no `&&`, no `@"..."@` with `$` vars unless escaped, no non-ASCII characters.
- The commit must happen AFTER `fw_version.h` is staged — the source commit used in `release.txt` should be the commit BEFORE the release commit (the source state), so read HEAD before committing.
